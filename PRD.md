# OnePanel PRD

## 1. 文档信息

- 产品名称：OnePanel
- 文档版本：v0.2
- 文档类型：MVP PRD
- 目标平台：macOS 14+
- 使用模式：纯本地单机使用

## 2. 产品概述

OnePanel 是一款面向 macOS 的轻量桌面记录工具，核心形态是一个可随时唤起的单面板编辑窗口。用户可通过系统状态栏或全局快捷键，在任意桌面快速打开同一个记录面板，随手查看、编辑、复制自己的常用命令、工作记录或临时文本。

## 3. 产品目标

- 让用户在不打断当前工作流的前提下，快速记录和取用常用文本。
- 提供一个足够轻、足够快、足够稳定的桌面记录入口。
- 第一版聚焦“一个面板 + 一份主文档 + 全局唤起 + 跨桌面可见”，降低学习成本和实现复杂度。

## 4. 目标用户

- 程序员：记录常用命令、脚本片段、调试步骤。
- 运维 / 测试：记录部署命令、排查指令、环境信息。
- 产品 / 设计 / 运营：记录临时文案、待办事项、常用备注。
- 所有希望在 macOS 桌面上快速记事、快速取用文本的用户。

## 5. 核心设计原则

- 只做一份主文档，不做文件夹、多文档、分类系统。
- 整个面板即编辑区，不提供单条记录的复制、删除、编辑按钮。
- 纯文本编辑，不支持 Markdown、富文本、附件。
- 纯本地单机使用，不做账号体系、云同步、协作。
- 优先保证“随时唤起、随手可写、随手可藏”。

## 6. MVP 范围

### 6.1 包含内容

- 系统状态栏图标入口。
- 点击状态栏图标显示/隐藏面板。
- 一个全局快捷键显示/隐藏面板。
- 默认快捷键为 `Command + Control + P`。
- 单一主文档的纯文本编辑。
- 内容自动保存到本地。
- 窗口支持拖动、自由调节大小。
- 窗口支持“悬浮 / 固定”两种状态。
- 面板焦点状态下按 `ESC` 隐藏。
- 支持跨 Mission Control 桌面唤起。
- 面板唤起后，切换桌面时始终可见。
- 记忆上次窗口大小、位置、固定状态。
- 最小设置页。

### 6.2 不包含内容

- Markdown 支持。
- 富文本编辑。
- 图片、附件、代码高亮。
- 搜索功能。
- 多文档、多标签、文件夹管理。
- 历史版本。
- 开机自启动。
- 云同步、导出同步、多人协作。

## 7. 核心使用场景

### 7.1 快速查看常用命令

用户正在任意应用中工作，通过快捷键唤起 OnePanel，快速查看一段常用命令，然后复制到终端或其他应用中使用。

### 7.2 临时补充工作记录

用户在处理任务过程中，随时将新的命令、备注或待办追加到 OnePanel 中，无需切换到完整文档应用。

### 7.3 跨桌面持续参考

用户唤起面板后，在切换不同桌面或不同应用时，面板仍保持可见，方便持续对照内容操作。

### 7.4 快速隐藏

用户完成查看或编辑后，可通过 `ESC` 或再次按快捷键立即隐藏面板，不打断当前工作节奏。

## 8. 用户故事

### 8.1 角色定义

- 高频文本记录用户：需要频繁记录和取用命令、文案、备忘的 macOS 用户。

### 8.2 用户故事列表

- 作为一名开发者，我希望通过全局快捷键快速打开记录面板，这样我不用切换到其他文档工具也能查看常用命令。
- 作为一名开发者，我希望面板在不同桌面之间持续可见，这样我在一边查内容、一边操作多个桌面时不会丢失参考面板。
- 作为一名用户，我希望整个面板就是一个可直接编辑的文本区，这样我不需要理解复杂的数据结构或记录模型。
- 作为一名用户，我希望关闭或隐藏面板后内容不会丢失，这样我可以把它当作长期维护的一页常用记录。
- 作为一名用户，我希望窗口大小、位置和固定状态被记住，这样每次打开都符合我的使用习惯。
- 作为一名用户，我希望按 `ESC` 就能快速隐藏面板，这样我可以在需要专注当前应用时立刻收起它。

## 9. 页面与窗口说明

### 9.1 状态栏入口

- 形态：macOS 状态栏图标。
- 作用：应用常驻入口，用于显示或隐藏 OnePanel 主面板。
- 交互：单击后切换主面板显示状态。

### 9.2 主面板窗口

- 形态：独立浮动窗口。
- 组成：标题区、固定按钮、设置入口、主文本编辑区。
- 作用：承载唯一主文档的查看与编辑。
- 交互重点：支持键盘输入、文本选择、复制、粘贴、拖动、缩放、ESC 隐藏。

### 9.3 设置页

- 形态：轻量设置窗口或设置面板。
- 作用：承载 MVP 必要配置项。
- 内容范围：快捷键设置、窗口默认行为、窗口状态记忆策略。

### 9.4 页面边界说明

- MVP 只有一个主编辑面板，不存在首页、列表页、详情页等多页面结构。
- 设置页不是主工作区，仅用于低频配置。

## 10. 功能需求

### 10.1 状态栏入口

#### 功能描述

应用启动后在 macOS 系统状态栏显示一个仅图标入口。用户点击图标时，可切换 OnePanel 面板的显示和隐藏状态。

#### 需求说明

- 状态栏图标始终显示。
- 图标样式为仅图标，不显示文字。
- 点击图标时，若面板未显示，则显示面板。
- 点击图标时，若面板已显示，则隐藏面板。

### 10.2 单面板记录

#### 功能描述

应用仅提供一个主记录面板。面板主体为纯文本编辑区域，用户可直接输入、修改、删除内容。

#### 需求说明

- 不拆分为多条记录或卡片。
- 不提供记录级操作按钮。
- 用户自行选择、复制、删除文本内容。
- 编辑区默认支持连续输入长文本。
- 应支持粘贴外部文本内容。

### 10.3 自动保存与本地存储

#### 功能描述

所有编辑内容自动保存到本地应用私有文件中，无需用户手动点击保存按钮。

#### 需求说明

- 采用应用私有本地文件存储。
- 默认对普通用户透明，不要求用户直接操作存储文件。
- 编辑后自动保存。
- 应用重启后可恢复上次保存内容。
- 数据不上传网络，不依赖云服务。

### 10.4 窗口行为

#### 功能描述

面板为独立浮动窗口，不采用 popover。窗口支持拖动位置、自由缩放尺寸，并记忆用户上次使用状态。

#### 需求说明

- 面板为独立窗口。
- 支持拖动窗口位置。
- 支持自由调整窗口大小。
- 记忆上次窗口大小。
- 记忆上次窗口位置。
- 记忆上次固定状态。

### 10.5 悬浮与固定

#### 功能描述

面板支持“悬浮”和“固定”两种窗口状态。

#### 状态定义

- 悬浮：普通浮动窗口，可被其他窗口遮挡。
- 固定：始终置顶，不被普通应用窗口遮挡。

#### 需求说明

- 面板右上角提供“固定 / 取消固定”按钮。
- 点击按钮后立即切换窗口状态。
- 固定状态在应用重启后需保持。

### 10.6 跨桌面显示

#### 功能描述

用户在任意 Mission Control 桌面，都可以通过状态栏图标或全局快捷键唤起面板。面板唤起后，应在当前桌面显示，并在用户切换桌面后继续保持可见。

#### 需求说明

- 在任意桌面均可触发面板显示。
- 面板显示位置跟随当前桌面上下文出现。
- 面板被唤起后，切换到其他桌面时，窗口仍持续可见。
- 该能力为核心体验要求，优先级高。

### 10.7 全局快捷键

#### 功能描述

支持一个全局快捷键，用于显示或隐藏 OnePanel 面板。

#### 需求说明

- 默认快捷键：`Command + Control + P`。
- 用户可在设置页中修改该快捷键。
- 快捷键为全局级别，不要求 OnePanel 处于前台。
- 当快捷键冲突或注册失败时，应给出明确反馈。

### 10.8 ESC 隐藏

#### 功能描述

当 OnePanel 面板处于焦点状态时，用户按下 `ESC`，面板立即隐藏。

#### 需求说明

- 仅在 OnePanel 窗口为当前焦点时生效。
- 隐藏不影响已保存内容。
- 再次通过状态栏或快捷键可恢复显示。

### 10.9 设置页

#### 功能描述

MVP 提供最小设置页，用于管理少量必要配置。

#### 设置范围

- 全局快捷键设置。
- 窗口默认行为相关设置。
- 是否记忆窗口状态。

#### 说明

- 开机自启动不纳入当前版本实现范围。
- 设置页应保持简洁，不扩展复杂偏好项。

## 11. 状态流转

### 11.1 核心状态

- 隐藏：面板未显示，但应用和状态栏入口仍存在。
- 显示-悬浮：面板显示，处于普通浮动状态，可被其他窗口遮挡。
- 显示-固定：面板显示，处于置顶状态，不被普通窗口遮挡。
- 设置中：设置页显示，可与主面板并存或独立显示。

### 11.2 主状态切换规则

- 隐藏 -> 显示-悬浮：用户通过状态栏或快捷键唤起面板，且上次状态为悬浮。
- 隐藏 -> 显示-固定：用户通过状态栏或快捷键唤起面板，且上次状态为固定。
- 显示-悬浮 -> 隐藏：用户点击状态栏图标、再次触发快捷键，或在面板聚焦时按 `ESC`。
- 显示-固定 -> 隐藏：用户点击状态栏图标、再次触发快捷键，或在面板聚焦时按 `ESC`。
- 显示-悬浮 -> 显示-固定：用户点击固定按钮。
- 显示-固定 -> 显示-悬浮：用户点击取消固定按钮。

### 11.3 跨桌面行为规则

- 面板从隐藏进入显示状态时，应出现在用户当前桌面。
- 面板显示后，若用户切换桌面，窗口应继续保持可见。
- 无论面板当前处于悬浮还是固定，只要处于显示状态，都必须满足跨桌面可见要求。

### 11.4 数据保存规则

- 用户编辑文本时自动保存。
- 隐藏面板不清空内容。
- 应用退出后再次启动，应恢复最近一次成功保存的内容和窗口状态。

## 12. 交互与 UI 要求

### 12.1 整体风格

- 简洁、轻量、克制。
- 视觉上更像“桌面工具”，而不是“完整文档系统”。
- 减少分栏、模块化结构，突出单面板输入体验。

### 12.2 结构要求

- 顶部区域包含应用标题和固定按钮。
- 主体区域为大面积纯文本编辑区。
- 不出现左侧导航、文件树、分类面板。

### 12.3 窗口体验

- 窗口尺寸需适应不同使用习惯。
- 调整大小和拖动应顺畅自然。
- 显示与隐藏动作应轻快、不拖沓。

## 13. 数据与状态

### 13.1 数据模型

- 仅一份主文档。
- 文档内容为纯文本字符串。

### 13.2 需要持久化的状态

- 主文档内容。
- 窗口位置。
- 窗口大小。
- 是否固定。
- 用户自定义快捷键配置。

### 13.3 建议的本地数据结构

- `documentContent`：主文档纯文本内容。
- `windowFrame`：窗口坐标与尺寸。
- `isPinned`：是否固定置顶。
- `hotkey`：用户当前快捷键配置。
- `rememberWindowState`：是否启用窗口状态记忆。

## 14. 异常场景

### 14.1 快捷键冲突

- 场景：用户设置了与系统或其他应用冲突的快捷键。
- 期望行为：保存前检测冲突或注册失败；给出明确提示；保留旧快捷键配置不被覆盖。

### 14.2 快捷键注册失败

- 场景：应用未能成功注册全局快捷键。
- 期望行为：提示用户当前快捷键不可用；状态栏入口仍可正常使用；引导用户重新设置。

### 14.3 数据保存失败

- 场景：本地文件写入失败、磁盘权限异常、磁盘空间不足。
- 期望行为：尽量保留内存中的当前编辑内容；提示保存失败；允许用户继续复制当前内容，避免数据立即丢失。

### 14.4 窗口位置异常

- 场景：用户外接显示器拔出后，历史窗口坐标落在不可见区域。
- 期望行为：下次唤起时自动将窗口纠正到当前主屏幕可见区域。

### 14.5 应用异常退出

- 场景：应用崩溃或系统强制关闭。
- 期望行为：下次启动时尽可能恢复最近一次自动保存内容；不要求恢复未落盘的全部字符，但需尽量降低丢失范围。

### 14.6 ESC 与文本输入冲突

- 场景：用户正在输入，但按下 `ESC`。
- 期望行为：只要 OnePanel 面板拥有焦点，`ESC` 优先执行隐藏窗口，不额外弹出确认框。

## 15. 技术实现建议

### 15.1 技术栈方向

- 推荐优先使用 SwiftUI 构建界面。
- 对于状态栏、窗口层级、跨桌面行为、全局快捷键等能力，允许混合使用 AppKit 能力补足。
- 建议采用“SwiftUI 负责界面，AppKit 负责窗口控制”的实现方式。

### 15.2 窗口实现建议

- 主面板建议使用独立 `NSWindow` 承载，而不是纯 `Popover`。
- 需要重点验证窗口的 `level` 与 `collectionBehavior` 设置。
- 相关设置至少需要满足三个目标：可从任意桌面唤起、显示后切换桌面仍持续可见、在固定与悬浮状态之间正确切换层级。

### 15.3 状态栏实现建议

- 使用 `MenuBarExtra` 或 `NSStatusItem` 作为状态栏入口。
- 若对点击行为和窗口控制要求更高，优先采用 `NSStatusItem` 自定义控制逻辑。

### 15.4 快捷键实现建议

- 全局快捷键建议使用成熟方案封装，如 Carbon HotKey 封装或常见 macOS 热键库。
- 设置页中应对快捷键录入、展示、冲突校验分别设计清晰逻辑。

### 15.5 数据存储建议

- MVP 可采用本地文件或 `UserDefaults + 文件` 的轻量组合方案。
- 主文档正文建议保存为单独文本文件，避免大段正文完全塞入 `UserDefaults`。
- 窗口状态和快捷键配置更适合使用轻量配置存储。

### 15.6 自动保存策略建议

- 可采用“编辑变更后短延迟保存”的方式，兼顾实时性与写盘频率。
- 应在应用进入后台、窗口隐藏、应用退出等关键时机补一次兜底保存。

### 15.7 测试建议

- 重点验证多桌面、多显示器、快捷键冲突、窗口恢复、固定状态切换等系统级行为。
- 重点做手动验收测试，因为窗口层级与桌面行为较难仅靠单元测试覆盖。

## 16. 验收标准

- 点击状态栏图标，可稳定显示 / 隐藏面板。
- 按 `Command + Control + P`，可在任意桌面显示 / 隐藏面板。
- 面板显示后切换桌面，窗口仍保持可见。
- 面板聚焦时按 `ESC`，窗口立即隐藏。
- 编辑内容自动保存，重启应用后内容仍存在。
- 调整窗口大小、位置、固定状态后，重新打开应用可恢复。
- 切换“固定”后，窗口置顶行为符合预期。
- 设置页可修改全局快捷键，并在下次使用时生效。

## 17. 成功标准

- 用户可在 1 秒级完成面板唤起。
- 用户能在任意桌面稳定访问同一个记录面板。
- 用户首次使用无需学习复杂结构，即可直接开始记录。
- 查看和复制常用命令的效率明显高于打开备忘录、文档或 IDE 文件。

## 18. 风险与注意事项

- macOS 全局快捷键注册可能与系统或其他应用产生冲突，需要提供冲突处理策略。
- “跨桌面唤起后持续可见”依赖窗口层级和 Collection Behavior 设置，属于实现重点。
- 自动保存需兼顾及时性与稳定性，避免频繁写盘造成体验问题。

## 19. 当前确认的产品边界

- 纯本地单机使用。
- 纯文本编辑。
- 单文档模式。
- 不支持搜索。
- 不支持多文档。
- 不支持历史版本。
- 不支持开机自启动。
- 默认快捷键为 `Command + Control + P`。
- 状态栏仅图标。
- 面板右上角提供固定 / 取消固定按钮。
